## 1. Introduction

### 1.1 What is Brain Hemorrhage?

A **brain hemorrhage** (also called intracranial hemorrhage) is bleeding inside the skull or brain tissue. Blood leaks from a ruptured blood vessel and accumulates, creating pressure that damages brain cells. It is a life-threatening medical emergency that requires immediate detection and treatment.

**Types of Brain Hemorrhage (covered in this project):**

| Type | Location | Appearance on CT |
|---|---|---|
| **Subdural** | Between brain surface and dura (outer membrane) | Crescent-shaped bright region along skull edge |
| **Intracerebral** | Within the brain tissue itself | Irregular bright blob inside brain matter |
| **Subarachnoid** | Between brain surface and arachnoid membrane | Bright streaks in brain sulci (folds) |

### 1.2 What is a CT Scan?

A **CT (Computed Tomography)** scan uses X-rays taken from multiple angles to create cross-sectional images of the brain. On a brain CT scan:
- **Dark areas** (low density) → Air, fluid (CSF in ventricles)
- **Gray areas** (medium density) → Brain tissue (gray and white matter)
- **Bright areas** (high density) → Bone (skull), fresh blood (hemorrhage)

**Why CT for hemorrhage detection?** CT is the first-line imaging tool for suspected brain hemorrhage because it's fast (~5 minutes), widely available, and fresh blood appears distinctly bright (hyperdense) on CT images.

### 1.3 Why Use Deep Learning / Machine Learning?

| Manual Detection | AI-Based Detection |
|---|---|
| Requires trained radiologist | Automated analysis |
| Takes 15-30 minutes per scan | Results in seconds |
| Subject to human fatigue | Consistent 24/7 performance |
| Shortage of radiologists in rural areas | Deployable anywhere |
| Can miss subtle bleeds | Learns from thousands of examples |

### 1.4 Project Objective

Build a system that:
1. Accepts a **brain CT scan image** as input
2. Uses a **CNN (Convolutional Neural Network)** to classify it as **Hemorrhage** or **Normal**
3. Compares CNN performance against traditional **ML models** (Random Forest, SVM, Logistic Regression)
4. Provides a web interface for image upload, prediction, history tracking, and analytics dashboard

## 3. Deep Learning Model (CNN)

### 3.1 What is CNN?

A **Convolutional Neural Network (CNN)** is a type of deep learning model designed specifically for processing images. Unlike traditional ML models that need manual feature extraction, CNNs automatically learn features from raw pixel data.

**CNN Key Operations:**

| Operation | What It Does | Analogy |
|---|---|---|
| **Convolution (Conv2d)** | Slides small filters across the image to detect patterns | Looking at the image through small windows |
| **ReLU (Activation)** | Removes negative values, keeps positive | Filters out irrelevant information |
| **MaxPooling** | Shrinks the image by keeping only the strongest signals | Zooming out to see the big picture |
| **Flatten** | Converts 2D feature maps into a 1D vector | Unrolling a grid into a list |
| **Linear (Dense)** | Fully connected layer for final classification | Making the decision based on all features |
| **Dropout** | Randomly disables 50% of neurons during training | Prevents memorization (overfitting) |
| **Sigmoid** | Outputs a probability between 0 and 1 | Final confidence score |

### 3.2 CNN Architecture (BrainCNN class)

```
Input Image: 1 x 128 x 128 (grayscale, resized from 224x224)
        │
        ▼
┌─────────────────────────────────────┐
│ Layer 1: Conv2d(1 → 32, 3x3) + ReLU + MaxPool(2x2)   │
│   Input: 1 x 128 x 128  →  Output: 32 x 64 x 64      │
│   Learns: Basic edges, lines, simple textures           │
└────────────────────┬────────────────┘
                     ▼
┌─────────────────────────────────────┐
│ Layer 2: Conv2d(32 → 64, 3x3) + ReLU + MaxPool(2x2)  │
│   Input: 32 x 64 x 64  →  Output: 64 x 32 x 32       │
│   Learns: Shapes, curves, texture patterns              │
└────────────────────┬────────────────┘
                     ▼
┌─────────────────────────────────────┐
│ Layer 3: Conv2d(64 → 128, 3x3) + ReLU + MaxPool(2x2) │
│   Input: 64 x 32 x 32  →  Output: 128 x 16 x 16      │
│   Learns: Brain structures, bright regions              │
└────────────────────┬────────────────┘
                     ▼
┌─────────────────────────────────────┐
│ Layer 4: Conv2d(128 → 256, 3x3) + ReLU + MaxPool(2x2)│
│   Input: 128 x 16 x 16  →  Output: 256 x 8 x 8       │
│   Learns: Hemorrhage patterns vs normal anatomy         │
└────────────────────┬────────────────┘
                     ▼
┌─────────────────────────────────────┐
│ Flatten: 256 x 8 x 8 = 16,384 values                   │
└────────────────────┬────────────────┘
                     ▼
┌─────────────────────────────────────┐
│ Linear(16384 → 256) + ReLU + Dropout(0.5)              │
│   Fully connected layer with regularization             │
└────────────────────┬────────────────┘
                     ▼
┌─────────────────────────────────────┐
│ Linear(256 → 1) + Sigmoid                               │
│   Output: Single probability value (0.0 to 1.0)         │
│   Near 0.0 → Hemorrhage    Near 1.0 → Normal            │
└─────────────────────────────────────┘
```

### 3.3 Understanding the Output

The CNN uses **binary classification** with sigmoid activation:

```
class_to_idx: { "hemorrhage": 0, "normal": 1 }  (alphabetical order)

Model Output (Sigmoid):
  0.02  →  Hemorrhage Detected  (confidence: 98%)
  0.15  →  Hemorrhage Detected  (confidence: 85%)
  0.85  →  Normal (No Hemorrhage)  (confidence: 85%)
  0.99  →  Normal (No Hemorrhage)  (confidence: 99%)
```

**Decision boundary:** Output < 0.5 → Hemorrhage, Output ≥ 0.5 → Normal

### 3.4 Training Process

```python
# Training configuration
IMG_SIZE = 128        # Images resized to 128x128
BATCH_SIZE = 32       # Process 32 images at a time
EPOCHS = 15           # Train for 15 complete passes through the dataset
Learning Rate = 0.001 # Step size for weight updates
Loss Function = BCELoss (Binary Cross-Entropy)
Optimizer = Adam      # Adaptive learning rate optimizer
```

**Training Loop (per epoch):**
1. Load a batch of 32 images with labels
2. Pass images through the CNN → get predictions
3. Calculate loss (how wrong the predictions are) using BCELoss
4. Backpropagate the error → compute gradients
5. Update weights using Adam optimizer
6. Repeat for all batches → 1 epoch complete
7. Repeat for 15 epochs

**BCELoss (Binary Cross-Entropy) Formula:**
```
Loss = -[y × log(p) + (1-y) × log(1-p)]

Where: y = actual label (0 or 1), p = predicted probability
```

If the model predicts 0.95 for a "normal" image (label=1): Loss = -[1 × log(0.95)] = 0.05 (low loss — good!)
If the model predicts 0.95 for a "hemorrhage" image (label=0): Loss = -[0 × log(0.95) + 1 × log(0.05)] = 3.0 (high loss — needs improvement!)

## 4. Traditional ML Models (Comparison)

### 4.1 Why Compare ML Models?

The project compares 4 approaches to show that **deep learning (CNN) outperforms traditional ML** for image classification tasks. This demonstrates why CNNs are the standard for medical imaging.

### 4.2 How ML Models Process Images

Unlike CNN which works directly on 2D images, traditional ML models need **flattened vectors**:

```
Original Image: 224 x 224 pixels
  ↓ Resize to 64 x 64
  ↓ Flatten (convert 2D grid to 1D list)
Feature Vector: [pixel_1, pixel_2, ..., pixel_4096]  (64 × 64 = 4,096 features)
```

**Key difference:** CNN learns hierarchical spatial features (edges → shapes → objects). ML models just see a flat list of pixel values — they lose all spatial information.

### 4.3 Models Compared

| Model | How It Works | Strengths | Weaknesses |
|---|---|---|---|
| **CNN** | Learns spatial features through convolution layers | Best for images, learns automatically | Needs GPU, more training time |
| **Random Forest** | 100 decision trees vote on the classification | Fast, handles noise well | Loses spatial info from flattening |
| **SVM** | Finds the best boundary (hyperplane) to separate classes | Good for binary classification | Slow on large feature sets |
| **Logistic Regression** | Linear model with sigmoid function | Simple, interpretable | Too simple for complex patterns |

### 4.4 Performance Results

| Model | Accuracy | Precision | Recall | F1 Score |
|---|---|---|---|---|
| **CNN (Deep Learning)** | **99.5%** | 99.01% | 100.0% | 99.5% |
| Random Forest | 92.0% | 87.5% | 98.0% | 92.45% |
| SVM | 86.5% | 79.2% | 99.0% | 88.0% |
| Logistic Regression | 76.5% | 71.54% | 88.0% | 78.92% |

### 4.5 Understanding the Metrics

**Confusion Matrix for CNN:**

```
                    Predicted
                 Hemorrhage  Normal
Actual Hemorrhage   99         1      ← 99 correct, 1 missed
Actual Normal        0       100      ← All 100 correct
```

| Metric | Formula | CNN Value | Meaning |
|---|---|---|---|
| **Accuracy** | (TP + TN) / Total | 99.5% | Overall correctness |
| **Precision** | TP / (TP + FP) | 99.01% | Of all "hemorrhage" predictions, how many were actually hemorrhage |
| **Recall** | TP / (TP + FN) | 100.0% | Of all actual hemorrhages, how many did we catch |
| **F1 Score** | 2 × (P × R) / (P + R) | 99.5% | Harmonic mean of precision and recall |

Where: TP = True Positive (correctly detected hemorrhage), TN = True Negative (correctly detected normal), FP = False Positive (normal misclassified as hemorrhage), FN = False Negative (hemorrhage missed)

**In medical context, Recall is critical:** A missed hemorrhage (FN) is far more dangerous than a false alarm (FP). The CNN achieves **100% recall** — it caught every single hemorrhage case.

#### Prediction Function — The Core Logic (Lines 133-160)

```python
def predict_image(image_path):
    # Step 1: Define preprocessing pipeline
    transform = transforms.Compose([
        transforms.Grayscale(num_output_channels=1),  # Ensure grayscale
        transforms.Resize((128, 128)),                 # Resize to model's expected input
        transforms.ToTensor(),                         # Convert to tensor (0.0-1.0)
    ])

    # Step 2: Load and preprocess the image
    img = Image.open(image_path).convert('RGB')   # Open uploaded image
    img_tensor = transform(img).unsqueeze(0)       # Add batch dimension: [1, 1, 128, 128]

    # Step 3: Run CNN inference
    with torch.no_grad():        # Disable gradient computation (saves memory)
        output = model(img_tensor)
        prob = output.item()     # Get single probability value

    # Step 4: Interpret the output
    # class_to_idx: hemorrhage=0, normal=1 (alphabetical)
    if prob < 0.5:
        prediction = 'Hemorrhage Detected'
        confidence = (1 - prob) * 100    # e.g., prob=0.02 → 98% confident
    else:
        prediction = 'Normal (No Hemorrhage)'
        confidence = prob * 100          # e.g., prob=0.99 → 99% confident

    return prediction, round(confidence, 2)
```

**Step-by-step flow:**
1. Image is loaded from disk using PIL
2. Converted to grayscale, resized to 128x128, normalized to 0-1 range
3. `.unsqueeze(0)` adds a batch dimension → shape becomes [1, 1, 128, 128]
4. `torch.no_grad()` disables gradient tracking (not needed for inference, saves memory)
5. Model outputs a single value between 0.0 and 1.0
6. Value < 0.5 → Hemorrhage, Value ≥ 0.5 → Normal
7. Confidence is calculated as distance from the decision boundary (0.5)

## 9. Key Concepts for Documentation

### 9.1 Deep Learning vs Machine Learning

| Aspect | Machine Learning | Deep Learning |
|---|---|---|
| Feature Extraction | Manual (human designs features) | Automatic (network learns features) |
| Data Requirement | Works with smaller datasets | Needs large datasets |
| Architecture | Flat (input → model → output) | Deep layers (many processing stages) |
| Image Processing | Requires flattening (loses spatial info) | Preserves spatial relationships |
| Best For | Structured/tabular data | Images, audio, text |
| Example in Project | Random Forest on 4096-pixel vector | CNN on 128x128 image |

### 9.2 Convolutional Neural Networks (CNN)

**Why CNNs for images?**
- **Local connectivity:** Each neuron only looks at a small region (3x3 filter)
- **Weight sharing:** Same filter applied across the entire image (translation invariant)
- **Hierarchical learning:** Lower layers learn edges, higher layers learn complex shapes
- **Parameter efficiency:** Much fewer parameters than a fully connected network

**CNN vs Fully Connected Network for a 128x128 image:**
- Fully connected: 128 × 128 × 256 = 4,194,304 parameters (first layer alone!)
- CNN: 3 × 3 × 1 × 32 = 288 parameters (first conv layer)

### 9.3 PyTorch Framework

| Concept | Explanation |
|---|---|
| `nn.Module` | Base class for all neural networks in PyTorch |
| `forward()` | Defines how data flows through the network |
| `model.train()` | Enables training mode (dropout active) |
| `model.eval()` | Enables evaluation mode (dropout disabled) |
| `torch.no_grad()` | Disables gradient computation (for inference) |
| `state_dict()` | Dictionary of model weights (for saving/loading) |
| `DataLoader` | Batches data and shuffles for training |
| `ImageFolder` | Auto-loads images from class-named folders |

### 9.4 Binary Classification with Sigmoid

```
Sigmoid function: σ(x) = 1 / (1 + e^(-x))

Input (any real number) → Output (0.0 to 1.0)

-5 → 0.007 (very likely class 0 = hemorrhage)
-2 → 0.119
 0 → 0.500 (uncertain — decision boundary)
+2 → 0.881
+5 → 0.993 (very likely class 1 = normal)
```

The last layer `Linear(256, 1)` produces a raw score, and `Sigmoid()` converts it to a probability.

### 9.6 Password Security (Werkzeug)

```python
# Registration: hash the password before storing
hashed = generate_password_hash('admin123')
# Result: 'pbkdf2:sha256:260000$abc123$def456...' (irreversible)

# Login: verify password against hash
check_password_hash(hashed, 'admin123')   # Returns True
check_password_hash(hashed, 'wrongpass')  # Returns False
```

**PBKDF2 (Password-Based Key Derivation Function 2):**
- Applies SHA-256 hash 260,000 times with a random salt
- Even if database is stolen, passwords cannot be recovered
- Each password has a unique salt, so identical passwords produce different hashes

## 10. Possible Viva/Exam Questions and Answers

**Q1: What is brain hemorrhage and how is it detected?**
A: Brain hemorrhage is bleeding inside the skull caused by ruptured blood vessels. It is detected using CT scans where fresh blood appears as bright (hyperdense) regions against the gray brain tissue. Our system automates this detection using a CNN deep learning model.

**Q2: Why did you use CNN instead of traditional ML models?**
A: CNN achieved 99.5% accuracy compared to Random Forest (92%), SVM (86.5%), and Logistic Regression (76.5%). CNNs process images directly and learn spatial features (edges, shapes, patterns) automatically, while ML models require flattening images into 1D vectors, losing all spatial information.

**Q3: Explain the CNN architecture used in this project.**
A: Our CNN has 4 convolutional layers (1→32→64→128→256 channels), each followed by ReLU activation and MaxPooling. After the conv layers, the feature maps are flattened into a 16,384-element vector, passed through a fully connected layer (16384→256) with dropout (0.5), and a final layer (256→1) with sigmoid activation that outputs a probability between 0 and 1.

**Q4: What is the role of each CNN layer?**
A: Layer 1 detects basic edges and lines. Layer 2 combines edges into shapes and curves. Layer 3 recognizes brain structures and bright regions. Layer 4 identifies hemorrhage-specific patterns. The classifier layers combine all learned features to make the final hemorrhage/normal decision.

**Q5: What is the difference between the 3 types of hemorrhage?**
A: Subdural hemorrhage occurs between the brain surface and outer membrane (crescent-shaped on CT). Intracerebral hemorrhage occurs within the brain tissue itself (irregular bright blob). Subarachnoid hemorrhage occurs between brain surface and middle membrane (bright streaks in brain folds).

**Q6: Why is the dataset synthetic and not real CT images?**
A: Real medical datasets require hospital permissions, ethical clearance, and patient consent. For educational demonstration, we generate synthetic images that mimic real CT scan characteristics — skull outline, brain tissue, ventricles, midline — with realistic hemorrhage patterns added.

**Q7: What preprocessing is done before feeding an image to the CNN?**
A: Three transformations: (1) Convert to grayscale (1 channel), (2) Resize to 128x128 pixels (CNN expects fixed input size), (3) Convert to PyTorch tensor (normalizes pixel values from 0-255 to 0.0-1.0). An additional `.unsqueeze(0)` adds a batch dimension.

**Q8: What is BCELoss and why is it used?**
A: BCELoss (Binary Cross-Entropy Loss) measures the difference between predicted probabilities and actual labels in binary classification. It penalizes confident wrong predictions heavily. For example, predicting 0.99 for a hemorrhage image (label=0) produces a very high loss, forcing the model to correct itself.

**Q9: How are passwords stored securely?**
A: Passwords are hashed using Werkzeug's PBKDF2-SHA256 algorithm with 260,000 iterations and a random salt. The original password cannot be recovered from the hash. During login, the submitted password is hashed and compared against the stored hash using `check_password_hash()`.

**Q10: What is the role of dropout in the CNN?**
A: Dropout(0.5) randomly disables 50% of neurons during each training step. This prevents the network from memorizing training data (overfitting) and forces it to learn more robust features. During prediction (`model.eval()`), dropout is disabled and all neurons are active.

**Q11: How does the model output get interpreted as hemorrhage or normal?**
A: The model outputs a single value between 0.0 and 1.0 via sigmoid. Since class indices are alphabetical (hemorrhage=0, normal=1), output < 0.5 means hemorrhage and output ≥ 0.5 means normal. Confidence is calculated as the distance from 0.5 — e.g., output 0.02 means 98% confidence in hemorrhage.

**Q12: Why is recall more important than precision in medical imaging?**
A: In medical diagnosis, a missed hemorrhage (false negative) is far more dangerous than a false alarm (false positive). High recall (100% for our CNN) means every hemorrhage is detected. A false positive only leads to additional testing, but a false negative could be fatal.

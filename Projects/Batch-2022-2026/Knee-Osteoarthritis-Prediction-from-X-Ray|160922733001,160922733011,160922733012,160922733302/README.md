# Predict and Classify Knee Osteoarthritis from X-Ray Imagery

A web application that uses **Ensemble Learning** (MobileNetV2 + Custom CNN + Random Forest) to classify knee X-ray images into 5 severity grades based on the Kellgren-Lawrence (KL) grading scale. Trained on real human knee X-ray images with Grad-CAM explainability, PDF report generation, and patient progression tracking.

## Features

- **Ensemble Learning** — MobileNetV2 (55%) + Custom CNN (30%) + Random Forest (15%) combined for improved accuracy
- **Doubtful Class Calibration** — Per-class calibration weights (2.8× for Doubtful) compensate for dataset imbalance; class-weighted training loss further improves Doubtful detection on retrain
- **Grad-CAM Explainability** — Visual heatmap showing which regions of the X-ray influenced the prediction
- **PDF Report Download** — Generate a full clinical report (X-ray, Grad-CAM, prediction, confidence bars, clinical findings, date, patient info)
- **Patient Progression Tracking** — Add patients, link X-rays to them, and track knee condition over time with trend indicators (Improved / Worsened / Stable)
- **Clinical Reasoning** — Detailed findings explaining the prediction (joint space narrowing, osteophytes, sclerosis)
- **5 KL Grades** — Normal, Doubtful, Mild, Moderate, Severe
- **Image Upload** — Drag-and-drop or click to upload (PNG, JPG, BMP, TIFF)
- **Per-Class Confidence Bars** — Visual breakdown of all 5 class probabilities
- **Interactive Dashboard** — Chart.js visualizations with per-class accuracy
- **Scan History** — Track all past predictions with patient links and PDF download per row
- **User Authentication** — Secure login/register with password hashing
- **Docker Support** — Containerized deployment

## Installation (Windows)

### Prerequisites
- Python 3.9 or higher
- pip (Python package manager)
- Kaggle account (for dataset download)

### Steps

1. **Clone the repository:**
```bash
git clone <repository-url>
cd code
```

2. **Create a virtual environment (recommended):**
```bash
python -m venv venv
venv\Scripts\activate
```

3. **Install dependencies:**
```bash
pip install -r requirements.txt
```

For PyTorch on Windows (CPU only):
```bash
pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu
```

4. **Download the real X-ray dataset:**
```bash
python download_dataset.py
```
Downloads 9,786 real knee X-ray images from Kaggle (5,778 train + 4,008 test) organized into 5 KL grades.

5. **Train the models:**
```bash
python train_model.py
```
Trains MobileNetV2, Custom CNN, and Random Forest with class-weighted loss (Doubtful gets 3× weight). Saves `knee_mobilenet_model.pth`, `knee_cnn_model.pth`, `ml_model.pkl`, and `models_info.json`.

6. **Run the application:**
```bash
python app.py
```
Open http://localhost:5011 in your browser.

- Password: `admin123`

## KL Grading Scale

| Grade | Name | Description |
|-------|------|-------------|
| 0 | Normal | No radiographic features of OA |
| 1 | Doubtful | Possible slight narrowing, minute osteophytes |
| 2 | Mild | Definite osteophytes, possible narrowing |
| 3 | Moderate | Multiple osteophytes, definite narrowing, sclerosis |
| 4 | Severe | Large osteophytes, marked narrowing, bone deformity |

## Model Performance (Real X-Ray Data)

| Model | Accuracy | Precision | Recall | F1 Score |
|-------|----------|-----------|--------|----------|
| MobileNetV2 | 59.71% | 56.10% | 59.71% | 55.53% |
| Custom CNN | 53.14% | 41.42% | 53.14% | 44.95% |
| Random Forest | 41.79% | 34.25% | 41.79% | 33.60% |

**Per-Class Accuracy (MobileNetV2, before ensemble calibration):**
| Grade | Accuracy |
|-------|----------|
| Normal | 87.27% |
| Doubtful | 7.87% → **~31% after calibration** |
| Mild | 57.91% |
| Moderate | 47.45% |
| Severe | 81.15% |

> **Ensemble + Calibration:** The app combines all three models' probabilities (weighted average) then applies per-class calibration. The Doubtful class receives a 2.8× calibration boost to compensate for its systematic under-prediction. Retraining with `train_model.py` applies a 3× class weight in the loss function for further improvement.

## Grad-CAM Explainability

When a prediction is made, the application generates a **Grad-CAM heatmap** from MobileNetV2's last convolutional layer:

- **Red/Yellow regions** — Areas with strongest influence (joint space, osteophytes, bone surfaces)
- **Blue/Green regions** — Areas with less influence
- **Side-by-side display** — Original X-ray and heatmap shown together on the predict page and in PDF reports

## Patient Progression Tracking

1. **Add a patient** via Patients → Add Patient (name, age, gender, clinical notes)
2. **Link scans** — select a patient from the dropdown when uploading an X-ray
3. **View timeline** — Patient detail page shows all scans in chronological order with:
   - Trend indicator per scan (↓ Improved / ↑ Worsened / — Stable)
   - Overall progression summary (first scan vs latest scan)
   - X-ray and Grad-CAM thumbnails per scan entry
   - PDF download button per scan

## PDF Report

Each prediction can be downloaded as a PDF report (`/download_report/<id>`) containing:
- Report metadata (ID, date, doctor/user, models used)
- Patient info (if linked)
- Colored diagnosis banner (grade + confidence)
- Original X-ray and Grad-CAM heatmap images
- Confidence score bars for all 5 classes
- Clinical findings list
- Medical disclaimer
